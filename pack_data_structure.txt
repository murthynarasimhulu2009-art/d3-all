      │   ]                         // end of root children array │

      │   depth: number,            // depth in the hierarchy (0 = root) │
      │   x: number,               // x‑coordinate of the packed circle │
      │   y: number,               // y‑coordinate of the packed circle │
      │   r: number                // radius of the packed circle │
      │ }                                                                     │
      └───────────────────────────────────────────────────────────────────────┘

Field explanations

- name – categorical identifier for each node (root, category, or item).  
- value – quantitative metric used to size circles (leaf nodes only).  
- children – nested array of child nodes; creates the hierarchical tree.  
- depth – numeric depth level added by the hierarchy transform (root = 0).  
- x, y – coordinates computed by the pack transform to position each circle.  
- r – radius computed from the summed `value` at each node, used for circle size.

The data is a single hierarchical tree (no separate links array). The `hierarchy` and `pack` transforms enrich each node with `depth`, `x`, `y`, and `r` for rendering the circle‑packing visualization.
