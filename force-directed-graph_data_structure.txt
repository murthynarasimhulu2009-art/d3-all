Explanation of fields:
- id – nominal identifier, uniquely distinguishes each node.
- group – nominal/categorical field used to assign colors (via the “color” ordinal scale).
- size – quantitative metric that could be mapped to visual size (not directly used in the current mark but available for custom encoding).
- name – textual label for the node, useful for tooltips or legends.
- source – identifier of the source node in a link (references a node’s id).
- target – identifier of the target node in a link (references a node’s id).
- value – numeric value representing link strength; used to compute stroke width (via sqrt transform).
- weight – numeric weight influencing the force‑directed layout (used in the force transform).

The data contains two nested arrays: nodes (array of node objects) and links (array of link objects). After the force transform, each node will also have generated x and y fields (quantitative coordinates) that are used by the “links” and “nodes” layers for positioning.
