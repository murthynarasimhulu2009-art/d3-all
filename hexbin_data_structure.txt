Explanation of fields:
- x – quantitative field representing the X‑axis value for each point; used by the linear x‑scale.
- y – quantitative field representing the Y‑axis value for each point; used by the linear y‑scale.
- count – quantitative field indicating how many original data points are aggregated into the hexbin cell; drives the sequential color scale.

The data does not contain nested arrays (no nodes/links). It is a flat array of objects, each representing a single observation that will be aggregated by the hexbin transform. The resulting hexbin geometry (the “path” field) is generated by the transform and used in the layer’s encoding.
